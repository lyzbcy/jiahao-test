## 批次 1：功能 5 · 表情包上线与深度互动

### 现状诊断（关键坑）
- `img/sticker/` 下有 190 张周三涵系列表情图，文件名为中文短句（`哈哈哈.png`/`不会吧.png`/`专注.png` 等）
- **被 `.gitignore` 第 15 行排除**，导致线上 GitHub Pages 根本加载不到这些图
- 代码中**零引用**，全程用 emoji 占位（😎🎯🎊）

### 设计
**① 表情包上线**
- 移除 `.gitignore` 对 `img/sticker/` 的排除，提交到仓库
- 新建 `img/sticker/index.json`：映射表情文件名 → 情绪标签（如 `{"专注.png":"focus","哈哈哈.png":"happy",...}`），供代码按场景取图

**② 做题反馈表情（互动点 1）**
- 提交选项/滑块后，根据豪意倾向弹对应表情包
- 配置：每个选项可带 `sticker` 字段指向某张图；未配置则按维度极性取默认表情

**③ 结果页本命嘉豪配图（互动点 2）**
- `labels.json` 每个豪型可带 `stickers` 数组（1-2 张契合表情）
- 结果页 `result.js` 在豪型卡上渲染这些表情做装饰，取代单一 emoji

**④ 排行榜/分享卡装饰（互动点 3）**
- 排行榜前三名用表情包做奖牌点缀
- 分享卡角落点缀一张情绪契合的表情包

**⑤ 吉祥物升级**
- 现吉祥物固定 emoji 😎，改为按做题阶段切换表情包图：闲置 `乖巧`、答题 `专注`、关键题思考 `不会吧`、出结果 `哈哈哈`/`享受美食`

## 批次 3

### 功能 2：豪意值小数化（方案 A）
- 上限保持 100，显示 `97.342` 三位小数
- `test-engine.js`：`calcHaoyi` 去掉 `Math.round`；时间加成/昵称加成浮点精确参与
- 档位判定基于小数值，边界值不变
- 显示层（result/ranking/share-card）统一 `toFixed(3)`
- 「击败全国 X%」基于小数重算百分位
- 数据库 `haoyi` 字段 `int` → `numeric(6,3)`（提供 SQL，表已清空无转换风险）

### 功能 3：7 级滑块题（方案 甲）
- 题库 `questions.json` 新增 `type` 字段：`"choice"`（现有）/ `"slider"`（新）
- 滑块题结构：`{ id, type:"slider", dim, text, scale:{min:1,max:7}, pole }`
- 新增 6 道纯程度自评滑块题（测四维 + 2 道迷惑性交叉题）
- 引擎 `score()` 增加分支：滑块 `score` = 用户选的 1-7 值，按 `pole` 累加，与 choice 题统一进入维度计分
- `home.js` `renderQuestion` 识别 `type===slider` 渲染 `<input type=range>` + 数值显示 + 确认按钮
- 滑块题穿插在 ABCD 题之间（打乱节奏防摸索）
